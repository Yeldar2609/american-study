## Final fix
The OneDrive path failure was resolved without a junction. `vitest.config.ts` already
carries `resolve.preserveSymlinks = true`, which is the necessary and sufficient fix: with
it, `npm test` runs green directly from
`C:\Users\yelda\OneDrive\Desktop\American Study App` — 24 files, 111 tests passing
(verified 2026-06-19). No junction, cache purge, or dependency change is required.

Root cause (confirmed): Vite/Vitest canonicalizes the OneDrive reparse-point path during
config/module resolution, so `setupFiles` and the `@` alias resolve away from the launch
directory. `preserveSymlinks` keeps the launch path intact and resolution succeeds.

Follow-on: `tests/e2e/admin-analytics.spec.ts` was the only file still failing lint and
typecheck (it predated any green gate run). Fixed by parsing `process.env` through the zod
schema directly (matching `src/lib/env.ts`) instead of cherry-picking keys, then letting
Biome reformat. lint + typecheck + test + build are now all green.

Not runnable on this machine (no Docker): `test:e2e`, `db:test` (pgTAP), `db:reset`, and
applying migrations. The e2e specs and pgTAP suites require a local Supabase stack.
